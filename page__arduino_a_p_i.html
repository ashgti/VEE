Arduino API and Control Interface

Arduino API

The Arduino is a software platform that can be built on top of various Atmel AVR processors. The emulator is based off of the Arduino platform targeting the Atmel AVR Atmega328p processor. The Atmega328p platform was selected because it is a commonly processor used in multiple Arduino devices. If a student want to try to apply their work to a real embedded system it should be possible to transfer their code to the Arduino by recompiling it to target that platform and not have to make any other changes to the code. The firmware for an Arduino is written in the C/C++ programming language. This software API abstracts away many of the implementation details of the hardware. The main strategy of the emulator is not to emulate the individual hardware components directly, but to implement the Arduino API.

Atmel AVR Atmega328p Hardware Abstractions

The Arduino API abstracts a number of hardware details from the end user. For instance, interrupts have been greatly simplified as well as other simple operations like reading from an UART. However, the Arduino API does not mask all hardware details from the developer. These hardware details can help teach students about embedded system constraints. The primary hardware abstractions the emulator still implements are the PORTx and DDRx registers. In the Atmega328p, the PORTx data registers hold the value of memory mapped IO pins, for doing digital reads and writes. The DDRx registers are used to control the direction of the pin, whether the pin is an input pin or an output pin.
