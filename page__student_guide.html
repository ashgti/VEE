Student Guide

Requirements

For a given project, the student should receive at least the following:

  • A clear objective of what the firmware should accomplish.
  • A configuration file provided by the professor.
  • The software libraries bundled as an Library Archive or the source files to allow them to compile the code into a single library.

Example Code

Students need to implement pinConfiguration, setup and loop in order for their code to function properly. Here is a very primitive example of registering a single signal to a pin:

void pinConfiguration() {
  registerPin("a", 1); // Registers Signal "a" to pin 1.
}

Once the pin is registered, it functions like a normal pin in the Arduino. Here is an example of configuring the pin to INPUT mode.

void setup() {
  pinMode(1, INPUT); // Configure the pin as an input line.
}

Finally, we can build the loop operation where we read a digital pin and dispatch a signal if the pin is HIGH.

void loop() {
  if (digitalRead(1) == HIGH) {
    processSignal("a"); // Dispatch a busy wait as a result of the pin going high.
  }
}

The end result would look like this:

void pinConfiguration() {
  registerPin("a", 1); // Registers Signal "a" to pin 1.
}

void setup() {
  pinMode(1, INPUT); // Configure the pin as an input line.
}

void loop() {
  if (digitalRead(1) == HIGH) {
    processSignal("a"); // Dispatch a busy wait as a result of the pin going high.
  }
}

Commandline Options

Use --help to have a complete list of all of the command line otions.

Usage: vee-cmd.py [options]

Options:
  -h, --help            show this help message and exit
  -f FILE, --file=FILE  write output to fileself
  -q, --quiet           don't print status messages to stdout
  -r RUNTIME, --runtime=RUNTIME
                        runtime length of the scenario
  -g, --graph           graph output of signals, requires matplotlib

The commandline comes with the option --graph which will try to graph the signal output.

Implementation Details

Students must implement the following:

  • pinConfiguration

    Important Must be implemented in the target Arduino Code.

  • setup

    Important Must be implemented for the Arduino code to work.

  • loop

    Important Must be implemented for the Arduino code to work.

The following API's are available, check arduino.h for a more complete list.

  • registerPin

    Map a pin to a signal.

  • registerSerial

    Map a signal to a serial port.

  • digitalRead

    Read a value off a given digital port.

  • digitalWrite

    Write to a pin a given value.

  • analogRead
  • analogWrite
  • DDRx Ports
    See also:
    DDRB DDRC DDRD
  • PORTx Ports
    See also:
    PORTB PORTC PORTD
  • HardwareSerial

    Hardware Serial represents the a basic hardware RS-232 Serial port.

  • HardwareSerial::begin

    Set the baud rate of the serial port.

  • HardwareSerial::available

    Returns the number of characters in input buffer.

  • HardwareSerial::read

    Return 1 character from the input buffer and removes that character from the buffer.

  • HardwareSerial::write

    Write a single character to the output buffer.

  • HardwareSerial::print
  • HardwareSerial::println

Students must map a given signal to a pin on the hardware board. For example, the signal "A" could be mapped to pin 4 using the registerPin() function.
